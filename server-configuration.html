21. write the procedure in installing DHCP server role and configure ​


Answer:

Sure, here's a general procedure for installing and configuring a DHCP server role on a Windows Server:

1. Open the Server Manager on your Windows Server.

2. Click on the "Add Roles and Features" option, which will start the installation wizard.

3. Select "Role-based or feature-based installation" and click "Next".

4. Choose the server you want to install the DHCP role on and click "Next".

5. Select the "DHCP Server" role from the list of available roles and click "Next".

6. Review the information on the "DHCP Server" screen and click "Next".

7. Click "Install" to begin the installation process.

8. Once the installation is complete, click "Close".

Now that the DHCP server role is installed, you can configure it by following these steps:

1. Open the DHCP management console from the Server Manager.

2. Right-click on the server name in the console tree and select "Authorize" to allow the DHCP server to operate in the Active Directory domain.

3. Right-click on the "IPv4" node in the console tree and select "New Scope" to create a new scope.

4. Follow the wizard to configure the scope, including the IP address range, subnet mask, default gateway, DNS server, and lease duration.

5. Once the scope is configured, right-click on it and select "Activate" to make it available to clients.

6. You can also configure additional DHCP options, such as DNS suffixes and WINS servers, by right-clicking on the server name and selecting "Set Predefined Options".

7. Finally, you can configure DHCP reservations for specific clients by right-clicking on the "Reservations" node and selecting "New Reservation". Here, you can specify the MAC address and IP address of the client.

That's it! Your DHCP server is now configured and ready to provide IP addresses to clients on your network.

Answer:

7 Common Network Issues and How to Resolve Them Fast

Networks are networks. Despite best efforts to keep things smooth all the time every day, things happen. Here’s a look at some common network issues, some tips for quickly resolving them, and even better, how to prevent them from occurring again.

1. Duplicate IP Addresses

When two devices attempt to share a single IP, you see the dreaded “Address Already in Use” Kill — with no ability to access the network.

The Quick Fix: The blame for this often rests with your router’s default DHCP configuration. DHCP is probably trying to assign your new device an address at the beginning of your subnet, and another device may already occupy these low-numbered addresses with static IPs. If you’ve just introduced a new device or server to your network, it may have its own DHCP server. Simply disable the DHCP server on that device to restore sanity to your network.

The Preventive Measure: You can take one simple step to avoid IP conflicts by modifying your router’s configuration to begin assigning DHCP addresses near the top end of your subnet, leaving the lower addresses available for devices that require static IPs.
